SHEET A-102 · COMPONENT VIEW

Technical Architecture

Frontend, backend, data stores and external providers, traced through a single chat-query request.

NTS8 Components
FIG A-102.1 — COMPONENT DIAGRAM

Component Inventory

ComponentTechnology
Frontend SPAReact 18, TypeScript, Vite, Tailwind
API GatewayFastAPI — validation, auth, routing
Schema LayerPydantic v2
Agent LayerLangGraph + LangChain, 7 agents
ORMSQLAlchemy → SQLite
Vector StoreChromaDB

Chat Query — Message Flow

  1. Frontend sends POST /api/chat
  2. Route validates schema, authenticates JWT
  3. Service loads conversation context
  4. Supervisor Agent invoked with query
  5. Routes to Knowledge Retriever → ChromaDB
  6. Chunks passed to downstream agents
  7. Response streamed via SSE
  8. Message + sources persisted

SHEET A-103 · ORCHESTRATION VIEW

Agent Workflow

Seven LangGraph agents route a user query from ingestion or retrieval through to a notified response.

NTS7 Agents
FIG A-103.1 — AGENT FLOWCHART

Agent Register

AgentTools
SupervisorRouter logic, SQLite conversation history
Doc AnalyzerMulti-format loader, OCR, NER
Knowledge RetrieverChromaDB retriever
Conflict DetectorLLM comparison prompt
SummarizationLLM summarization prompt
RecommendationLLM reasoning prompt
NotificationNotification service

Error Handling — BR-5

LLM calls retry with exponential backoff, max 3 attempts. Irrecoverable failures fall back gracefully and log to audit_logs.

Conflict Detector and Recommendation require confidence ≥ 0.6; below threshold, output is withheld for manual review.
